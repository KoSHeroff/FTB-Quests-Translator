# Переводчик квестов FTB Quests (SNBT → русский)

Скрипт читает папку с файлами квестов в формате SNBT (как в модпаках Minecraft с **FTB Quests**), отправляет английские строки в API совместимый с OpenAI и сохраняет переведённые файлы в другую папку. Повторные запуски используют **кэш**, чтобы не платить за уже переведённый текст.

Никакого отдельного «сервиса» в браузере нет: это **локальная программа на Python**, которую вы запускаете у себя на компьютере.

---

## Что нужно заранее

1. **Python 3.10+** — [python.org](https://www.python.org/downloads/) (при установке отметьте «Add Python to PATH» на Windows).
2. **Аккаунт и ключ API** у OpenAI или у провайдера с **совместимым** API (часто тот же формат запросов, что у OpenAI).
3. Папка с квестами — обычно что-то вроде `config/ftbquests/quests` из мода/модпака (файлы `.snbt`).

---

## Установка по шагам

### 1. Скачайте или клонируйте этот проект

Рабочая папка — там, где лежат `translate_quests.py` и `requirements.txt`.

### 2. Виртуальное окружение (рекомендуется)

В терминале **в папке проекта**:

**Windows (PowerShell или cmd):**

```text
python -m venv .venv
.venv\Scripts\activate
```

**Linux / macOS:**

```text
python3 -m venv .venv
source .venv/bin/activate
```

После активации в начале строки терминала обычно появляется `(.venv)`.

### 3. Зависимости

```text
pip install -r requirements.txt
```

### 4. Файл настроек `.env`

1. Скопируйте `.env.example` в файл с именем **`.env`** (без `.example`).
2. Откройте `.env` в блокноте и вставьте свой ключ:

   - `OPENAI_API_KEY` — строка ключа (часто начинается с `sk-...`).

3. При необходимости измените:
   - `OPENAI_MODEL` — например `gpt-4o-mini` (дешевле) или другая модель, которую поддерживает ваш провайдер.
   - `OPENAI_BASE_URL` — **оставьте пустым** для обычного OpenAI. Заполняйте только если используете **другой адрес API** (прокси, Azure OpenAI, локальный сервер с совместимым API). Значение — полный базовый URL, обычно заканчивается на `/v1` (как в документации вашего провайдера).

**Важно:** не выкладывайте `.env` в интернет и не коммитьте в git — в нём секретный ключ.

## Запуск

Команды выполняйте **из папки проекта**, с активированным `.venv`, если вы его создавали.

### Обычный перевод

Входная папка — где лежат `.snbt`, выходная — куда записать результат (папку можно не создавать заранее):

```text
python translate_quests.py --input путь\к\quests --output путь\к\quests_ru
```

По умолчанию, если не указывать `--input` / `--output`, используются папки `quests` и `quests_ru` **относительно текущей директории** в терминале.

Кэш переводов по умолчанию сохраняется в **`translation_cache.json`** рядом со скриптом (не зависит от того, из какой папки вы вызвали `python`).

### Полезные режимы

- **Проверка без API** — только собрать/посмотреть объём строк:

  ```text
  python translate_quests.py --dry-run
  ```

- **Сравнить id квестов** во входной и выходной папках (без перевода):

  ```text
  python translate_quests.py --verify-only --input quests --output quests_ru
  ```

- **Подробный лог** в консоль:

  ```text
  python translate_quests.py -v --input quests --output quests_ru
  ```

Полный список опций:

```text
python translate_quests.py --help
```

## Что делать с результатом

Скопируйте содержимое выходной папки (`quests_ru` или та, что указали в `--output`) обратно в модпак вместо оригинальных квестов (сохраните резервную копию оригинала). Точный путь зависит от сборки; часто это снова `config/ftbquests/quests`.

---

## Частые проблемы

| Симптом | Что проверить |
|--------|----------------|
| `OPENAI_API_KEY` не задан | Файл `.env` в корне проекта, ключ без лишних кавычек, после правок перезапустите команду. |
| Ошибки подключения / 401 | Ключ, `OPENAI_BASE_URL` и модель соответствуют вашему провайдеру. |
| «Не та папка» | Укажите полные пути в `--input` и `--output` или перейдите в нужную папку через `cd` перед запуском. |

---

## Кратко

1. Установить Python → `python -m venv .venv` → активировать → `pip install -r requirements.txt`.
2. Скопировать `.env.example` → `.env`, вписать `OPENAI_API_KEY`, при необходимости `OPENAI_MODEL` и `OPENAI_BASE_URL`.
3. Запустить: `python translate_quests.py --input <квесты> --output <куда сохранить>`.
